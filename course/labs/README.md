# Labs — hands-on

Reading about overfitting doesn't change how you trade. *Catching a mirage with your own hands* does.
These labs are turnkey: bundled synthetic-but-realistic data, no downloads, no API keys. Run them in
order, then point the capstone at your own strategy.

## Setup

```bash
pip install overfitguard        # or: pip install -e .  from the repo root
cd course/labs
python make_data.py             # generates candidates.csv for Lab 3 (prices/your_strategy come bundled)
```

## The labs

| Lab | Chapter | What you'll do |
|---|---|---|
| `lab1_the_mirage.py` | 1 | Manufacture a beautiful backtest from **pure noise**, then watch OverfitGuard expose it. |
| `lab2_four_defenses.py` | 2 | Put an honest moving-average strategy through all **four defenses**, net of costs, vs a benchmark. |
| `lab3_the_screener.py` | 2–3 | Run **White's Reality Check** on 250 candidates and find the one real edge buried in the noise. |
| **`capstone.py`** | all | **Audit your OWN strategy.** The whole course, in one command. |

## The capstone — the point of all of this

```bash
python capstone.py                          # audit the sample (a mirage — watch it fail)
python capstone.py your_returns.csv 300     # audit YOUR returns, honest about how many configs you tried
```

Your CSV needs a column of periodic net-of-cost returns (a `date` column is optional). You'll get a
verdict, a shareable HTML report, and a plain-English read on whether you've *earned the right to
believe it.* Most strategies — including ones that felt like discoveries — do not. That knowledge, on
your own strategy, before you risk money on it, is the single most valuable thing in this course.

## The data (`data/`)

- `prices.csv` — a synthetic total-return "index" (the honest benchmark). *(bundled)*
- `your_strategy.csv` — a sample strategy that looks brilliant in-sample and dies out of it. *(bundled)*
- `candidates.csv` — a 250-strategy search with exactly one real edge hidden inside. *(generated by
  `make_data.py` — too large to bundle)*

All deterministic and regenerable via `make_data.py`. Swap in real data whenever you like — the tools
only need returns.
